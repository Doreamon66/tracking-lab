# 模板Sheet结构详细定义

---

## 一、模板对比

| 特征 | 位置模型模板 | 通用模型模板 |
|------|------------|------------|
| 埋点方案列数 | 12列（A-L） | 11列（A-K） |
| 位置维度 | 页面中文 + 模块中文 + 元素中文 | 一级分类 + 二级分类 |
| 合并列 | A/B/C/L 跨行合并 | A/B/K 跨行合并 |
| 位置属性 | 有（klm、module_name等） | 无 |
| 适用场景 | UI结构固定、需标准化位置追踪 | 交互灵活、需自定义事件 |

---

## 二、基本信息 Sheet

两个模板结构相同。

```
A1:C1  合并 — 埋点结构说明（保留原内容）
A2     分类层级 | B2 中文名称 | C2 英文标识
A3     应用归属 | B3 {中文}   | C3 {英文编码}
A4     应用终端 | B4 {中文}   | C4 {英文编码}
A5     业务归属 | B5 {中文}   | C5 {英文编码}
A6     功能名称 | B6 {中文名} | C6 {功能标识}
A7     需求文档 | B7 {文档名} | C7 {文档URL}
```

填充规则：
- A1:C1 合并区域保留原始描述文字（"埋点结构：事件+ 属性，属性由公共属性+位置属性+自定义属性构成..."）
- B3:C6 由第1步用户选择的中文和对应的英文编码填充
- C3/C4/C5 英文编码从MCP返回的选项列表中提取
- B7 填入用户提供的云文档链接对应的文档中文名称
- C7 填入用户提供的云文档完整 URL

---

## 三、公共属性 Sheet

两个模板结构相同，表头6列：属性分类(A) | 属性名(B) | 属性含义(C) | 属性值格式(D) | 属性值描述(E) | 属性值备注(F)

### 位置模型模板 — 公共属性分区

| 分区 | 属性 |
|------|------|
| 公共属性 | entry_id、page_name、func_version、page_position、element_type |
| 位置属性 | page_name、page_position、module_name、module_position、second_module_name、second_module_position、element_name、element_position、element_type、klm |
| 入口属性 | first_entry、second_entry |
| 资源相关属性 | rid、resource_name、resource_key、sub_kind |
| 搜索相关属性 | keyword、tag_name、search_type |
| 精准相关属性 | recommend_id、policy |
| 链路追踪属性 | track_id、is_track_start |

> 注意：位置属性分区中的 page_name/element_type 与公共属性分区有重叠，在不同事件中根据需要引用。

### 通用模型模板 — 公共属性

| 分区 | 属性 |
|------|------|
| 公共属性 | func_version、is_login、member_identity、entry_id |

> **重要**：以上四项为**无标准业务**使用通用模型时的默认公共属性。  
> **AI / 增值 / PDF / 稻壳 / 图片** 等有业务规范时，**不得**套用本表；须按 [attribute-rules.md](attribute-rules.md) §5 填写该业务规范定义的公共属性。

## 四、埋点方案 Sheet

### 位置模型模板（12列）

**表头**（Row 1-2）：
```
A1:L1 合并 — 埋点位置（左半） | D1:L1 合并 — 事件信息（右半）
A2:页面中文 | B2:模块中文 | C2:元素中文 | D2:事件名称 | E2:事件中文 | F2:属性名称 | G2:属性中文 | H2:属性类型 | I2:属性值 | J2:属性值描述 | K2:属性值备注 | L2:事件触发时机
```

**数据区规则**：
- 每个事件至少包含一行事件头（D列事件名称 + L列触发时机）
- 属性行从 F 列开始填写（F~K）
- **合并规则**：
  - A列（页面中文）：同一页面的所有行合并
  - B列（模块中文）：同一页面下同模块的所有行合并
  - C列（元素中文）：同一模块下同元素的所有行合并
  - L列（事件触发时机）：同一事件的所有属性行合并
- **页面级事件**：模块填 `$页面级事件`，元素填 `--`
- **公共模块**：模块中文名称前加 `*`（如 `*搜索`、`*资源列表`、`*猜你喜欢`）

**每个事件的标准公共属性行**（固定顺序）：
1. event_id — 事件ID — 公共属性
2. page_name — 页面名称 — 公共属性 — 值={页面英文名}
3. module_name — 模块名称 — 公共属性 — 值={模块英文名}
4. element_name — 元素名称 — 公共属性 — 值={元素英文名}
5. element_type — 元素类型 — 公共属性 — 值=page/button/input/tag/card等
6. klm — 点位信息 — 公共属性
7. func_version — 功能版本 — 公共属性
8. is_login — 是否登录 — 公共属性
9. member_identity — 会员身份 — 公共属性
10. entry_id — 入口标识 — 公共属性

自定义属性追加在公共属性之后。

### 通用模型模板（11列）

**表头**（Row 1-2）：
```
A1:B2 合并 — 一级分类（左上） | C1:K1 合并 — 事件信息（右侧）
A2:一级分类 | B2:二级分类 | C2:事件名称 | D2:事件中文 | E2:属性名称 | F2:属性中文 | G2:属性类型 | H2:属性值 | I2:属性值描述 | J2:属性值备注 | K2:事件触发时机
```

**数据区规则**：
- 每个事件至少包含一行事件头（C列事件名称 + K列触发时机）
- 属性行从 E 列开始填写（E~J）
- **一事件一块**（硬性，详见 [attribute-rules.md](attribute-rules.md) §2.3）：
  - **同一事件名称（C 列）在全表只出现 1 次** = 1 个事件块；块内多行属性（公共 + 自定义）正常
  - 展示/点击/功能展示/功能点击类事件不得按页面、按钮、模块拆成多个同名块；场景写入 H 列属性值枚举
- **合并规则**：
  - A列（一级分类）：同分类所有行合并
  - B列（二级分类）：同分类下同二级所有行合并
  - K列（事件触发时机）：同一事件的所有属性行合并
- 一级分类 = 功能模块或页面名
- 二级分类 = 子模块、事件级别或分组标识

**每个事件的标准公共属性行**（固定顺序）：
1. func_version — 功能版本 — 公共属性
2. is_login — 是否登录 — 公共属性
3. member_identity — 会员身份 — 公共属性
4. entry_id — 入口标识 — 公共属性

> 上表仅适用于**无标准业务 + 通用模型**。有业务规范时按规范定义公共属性行（如 AI 业务为 `project/comp`、`open_position` 等，见 [attribute-rules.md](attribute-rules.md) §5）。

自定义属性追加在公共属性之后。

## 五、样式规范

| 元素 | 样式 |
|------|------|
| 表头行 | 微软雅黑 11pt 粗体；填充 #C5E0B4；水平居中+垂直居中；边框 |
| 数据行 | 微软雅黑 10.5pt；垂直居中+自动换行；边框 |
| 属性值列 | 有枚举值的属性直接填英文枚举值（多个值用 `/` 分隔，如 `web/client/noop`、`success/fail`、`true/false`）；无枚举值的属性填格式标注：`string`（默认）、`bigint`、`boolean`、`timestamp` |
| 属性值描述列 | 有枚举值时填写对应的中文释义（如 `PC端/Mac端`、`成功/失败`）；无枚举值时填写业务描述 |
| 属性类型 | "公共属性" 或 "自定义属性" |

---

## 六、实现建议

**推荐方式**：复制 `assets/` 下的模板文件，打开副本修改数据（保留合并单元格、样式、列宽）。

**注意事项**：
- openpyxl 操作合并单元格时先解除旧合并再创建新合并，避免重叠冲突
- 位置模型的公共属性Sheet分区较多，更新时按分区整体替换
- 通用模型的公共属性Sheet较简洁，根据业务规范追加属性
